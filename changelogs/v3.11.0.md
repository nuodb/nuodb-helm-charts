
# Changelog [v3.11.0](https://github.com/nuodb/nuodb-helm-charts/tree/v3.11.0) (2025-10-31)

## [Full Changelog](https://github.com/nuodb/nuodb-helm-charts/compare/v3.10.0...v3.11.0)

## New features

- Added PodMonitor resource for database pods to publish metrics to Prometheus.
- Added HorizontalPodAutoscaler (HPA) resources to auto-scale TEs based on metrics.
- Added ScaledObject resources to auto-scale TEs based on events using KEDA.
- Added the ability to auto-generate the password for the DBA user for a database.
- Added the ability for users to inject their own scripts to prepare the archive and journal directories for an SM to enable importing of database data.

## Resolved issues

- Previously, it was possible for SMs or TEs to listen on the incorrect port if they were unable to bind to port `48006` for any reason, which would cause them to be unreachable. This was fixed by specifying `portRange` as `48006,48006`, which defines an explicit range consisting only of port `48006`.
